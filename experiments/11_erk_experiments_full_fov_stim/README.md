# ERK Full-FOV Stimulation Experiments

## Notebooks

### `stim_rtmsequence.ipynb` -- RTMSequence API (recommended)

Runs a multi-phase ERK-KTR full-FOV stimulation experiment using the **RTMSequence API**. Phases are defined declaratively with `RTMSequence` objects (time plan, channels, stim_frames, ref_frames) and concatenated with `+`. This is the most concise way to set up an experiment.

### `stim_dfacquire.ipynb` -- df_acquire API

Functionally equivalent to `stim_rtmsequence.ipynb` but uses the **legacy `df_acquire` DataFrame API**. Acquisition events are built as a pandas DataFrame via `generate_df_acquire()` + `apply_stim_treatments_to_df_acquire()`, then converted to RTMEvents with `df_to_events()`. Useful as a reference for migrating older experiments.

### `stim_ramp_dfacquire.ipynb` -- Multi-phase drug test (df_acquire API)

A more complex experiment using the df_acquire API: 3 FOV groups with different stimulation treatments, each split into pre-drug and post-drug phases. Drug is pipetted manually between phases. Demonstrates `StimTreatment` with ramp-up exposures, `split_df_acquire()` for multi-phase scheduling, and `continue_experiment()` for preserving pipeline state across phases.

## Data Viewer

`DataViewer_ERK.py` is a standalone napari-based tool for visualizing and analyzing ERK-KTR experiment data. It loads legacy TIFF datasets, overlays segmentation masks and stimulation readouts, and plots per-cell ERK-KTR ratio (CNR) traces interactively.

### Features

- Browse experiment folders and load FOVs from OME-Zarr or TIFF storage
- Overlay raw images, stimulation readout, segmentation labels, and cytoplasmic rings
- Click on a cell in the viewer to plot its ERK-KTR ratio time course
- Filter and compare traces by treatment, drug concentration, or stimulation pattern
- Normalize CNR traces to a baseline window

### Usage

From the repository root:

```
uv run experiments/11_erk_experiments_full_fov_stim/DataViewer_ERK.py
```

The script uses [PEP 723](https://peps.python.org/pep-0723/) inline metadata, so `uv run` automatically installs the required dependencies (napari, pandas, zarr, etc.) into an isolated environment. No manual setup is needed.

### Data requirements

The viewer expects an experiment directory containing:

- `raw/`, `stim/`, `mask/`, etc. legacy per-FOV TIFF folders
- `exp_data.parquet` -- merged per-cell tracks with columns like `cnr`, `timestep`, `fov`, `particle`, `stim_exposure`, etc. (generated by `utils.generate_exp_data_from_tracks()` after acquisition)
